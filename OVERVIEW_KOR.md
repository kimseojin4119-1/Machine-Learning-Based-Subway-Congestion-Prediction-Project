# [교통/ML] 지하철 승차량 예측을 통한 지하철 배차 최적화 프로젝트

- 두 가지 ML 모델 탐색, XGBoost + RF 앙상블 모델 최종 선택

## 모델 비교

| Model | Train MAE | Test MAE | 총평 |
|---|---|---|---|
| **XGBoost+RF 앙상블 ✅** | 6.66 | **9.51** | LSTM 대비 낮은 오차, 정적 피처 중심 데이터에 적합 |
| LSTM | 24.94 | 28.59 | 시계열 모델이지만 이 데이터셋에서는 상대적으로 오차가 큼 |

> 위치·시설·날씨 등 정적 피처 비중이 커서, 시계열 패턴에 강한 LSTM보다 
> 트리 기반 앙상블 모델이 더 낮은 오차를 보였음

## 🚇 개요
- 문제 정의 :
  - 지하철 혼잡도를 사전에 예측하면 배차 조정, 안전 관리 등에 활용할 수 있다.
  - <img width="1280" height="500" alt="image" src="https://github.com/user-attachments/assets/543b9548-2d5f-414d-bffd-a54f125dc16f" />
  - 실제로 2023년 3분기 서울 지하철의 경우 4호선과 7호선의 혼잡도가 매우 높아 안전 관리 문제와 배차 간격 최적화 문제가 대두되었다.
  - 이와 같은 문제점을 바탕으로 지하철 승차 데이터를 활용하여 지하철 승차량을 예측하고 인사이트를 도출한다.
- 프로젝트 목표 :
  - 2024년 12월 1일 하루동안의 시간대별 지하철 승차 인원 예측
- 기간 및 인원 :
  - 2024.10.25 ~ 2024.11.30 (1개월) / 4명
- 예상 결과물 :
  - 시간대별 승차 인원 예측 모델
  - 모델별 성능 비교 및 최적 모델 선정
  - 해당 모델 결과에 따른 배차 최적화 결론
- 주요 역할 :
  - 데이터 수집 및 전처리
  - 머신러닝 모델링 (XGBoost, LSTM)
  - 모델 해석 (두 모델의 결과 비교)
  -배차 최적화 결론 해석 (정량 및 정성적 해석)
- 주관 :
  - 대전교통공사

## 🧹 데이터 수집 및 전처리
- 데이터 수집
- 주요 데이터 :
  - 지하철 승하차 데이터로서 2020.01 ~ 2024.10월까지의 일별 데이터로 대전 지하철의 판암부터 반석까지 모든 지하철 역의 데이터가 제공되었음
  - 0시부터 23시까지의 시간대별로 데이터 제공됨
  - 대전교통공사에서 직접 데이터를 제공하여 데이터의 공신력이 높음
  - 2024.11월 데이터는 공공데이터포털에서 제공함
  - [데이터 출처 링크](https://www.data.go.kr/)
- 추가 데이터 :
  - 지하철 승차량에 영향을 주는 요인으로 설명하기 위해 추가 데이터를 구하여 모델의 예측에 사용함
  - 추가 데이터는 대전 버스 승하차 데이터, 버스요금 데이터, 버스-지하철 환승 데이터, 지하철 역 500m 근방 병원, 마트, 교회, 영화관, 카페 데이터,
  - 역 출입구 데이터, 지하철 역 500m 유동인구 데이터, 동 인구 데이터, 강수량 및 적설 데이터 활용함
  - 추가 데이터는 각각 순서대로 대전 교통 빅데이터 플랫폼, 대전광역시청, 네이버 지도, 소상공인 상권분석시스템, 대전 각 구청, 기상청에서 데이터 수집함
- 데이터 전처리
- 전처리 과정 :
  - STEP 1. 승차 데이터만 사용하였으며 2024년 12월 1일의 날짜가 일요일인 주말이므로 평일 데이터를 제외한 주말 데이터만 남기고 삭제 처리하였으며 주말이지만 공휴일인 경우 승차량이 비정상적으로 높게 나와 삭제 처리
  - STEP 2. 동 인구
